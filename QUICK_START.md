# Быстрый старт для тестирования с несколькими пользователями

## Шаг 1: Запустите WebSocket сервер

В первом терминале:
```bash
npm run server
```

Или:
```bash
node server-example.js
```

Вы должны увидеть: `WebSocket server running on port 3001`

## Шаг 2: Создайте публичный туннель

Во втором терминале:
```bash
npm run tunnel
```

Или:
```bash
lt --port 3001
```

Вы получите публичный URL, например:
```
your url is: https://random-name.loca.lt
```

## Шаг 3: Обновите адрес сервера в приложении

### Вариант A: Через переменную окружения

Создайте файл `.env` в корне проекта:
```env
VITE_WS_SERVER_URL=https://random-name.loca.lt
```

### Вариант B: Временно в коде

Откройте `src/components/PixelStreamingViewer.vue` и измените:
```javascript
const wsServerUrl = "https://random-name.loca.lt";
```

## Шаг 4: Перезапустите приложение

```bash
npm run dev
```

## Шаг 5: Поделитесь URL с другими

Теперь другие пользователи могут:
1. Открыть ваше приложение (если оно тоже доступно публично)
2. Или использовать тот же туннель для фронтенда

## Альтернатива: Используйте ngrok (более стабильный)

1. Зарегистрируйтесь на https://ngrok.com
2. Установите ngrok
3. Запустите: `ngrok http 3001`
4. Используйте полученный URL

## Важно:

- ⚠️ localtunnel URL меняется при каждом запуске
- ⚠️ Для постоянного URL используйте ngrok с бесплатным планом
- ⚠️ Убедитесь, что сервер запущен перед созданием туннеля
