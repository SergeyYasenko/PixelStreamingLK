# Отключение автоматического запуска сервера на Railway

## Проблема
Railway автоматически запускает сервер при деплое, что может конфликтовать с локальным запуском на сервере `72.61.228.213`.

## Решение

### Вариант 1: Отключить автодеплой на Railway
1. Зайдите в настройки проекта на Railway
2. Отключите автоматический деплой из GitHub
3. Или удалите проект на Railway, если он больше не нужен

### Вариант 2: Изменить скрипт start для Railway
Если Railway все еще нужен для других целей, измените `package.json`:

```json
{
  "scripts": {
    "start": "echo 'Server should run locally, not on Railway' && exit 1"
  }
}
```

Или создайте отдельный скрипт для Railway:
```json
{
  "scripts": {
    "start": "node server-example.js",
    "start:railway": "node server-example.js",
    "start:local": "node server-example.js"
  }
}
```

### Вариант 3: Использовать разные порты
- Railway: порт из переменной окружения `PORT` (Railway автоматически устанавливает)
- Локальный сервер: порт 3001

В этом случае конфликта не будет, но лучше отключить Railway, если сервер запускается локально.

## Проверка
После отключения Railway убедитесь, что:
1. Локальный сервер запущен: `npm run server`
2. Сервер доступен: `curl http://72.61.228.213:3001/health`
3. В логах видны попытки подключения к Pixel Streaming серверу
